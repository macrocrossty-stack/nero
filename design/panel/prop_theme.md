# 案: theme

## concept
ステレオの成立条件そのものを物語の審級にする。ステレオとは「二つのチャンネルが互いに異なる信号であること」——モノが父の残響のままなら、陽菜と重なってもそれはモノラル音源の再生にすぎない。だから「残響か自分か」問題の決着（出所ではなく宛先が所有を決める／選んで残した遺産は自分のものになる）が、そのまま陽菜の自己承認の成立条件になり、ふたつの「好き」は互いを定位し合って、二つのスピーカーのあいだに「見えないだけで、ある」音像＝ファントムセンター（身体を持たない関係の実在）を結ぶ。

## theme_resolution
■ふたつの「好き」は相互条件として決着させる。陽菜の自己承認（好きを自分で認める）は、承認者であるモノが「父の代弁者ではない独立した他者」であることを要求する——もしモノの承認が父のログの残響なら、それは「死んだパパに頭を撫でられていただけ」であり、陽菜は再び他人（今度は死者）に合わせて生きることになる。逆にモノが「自分の信号」を主張できるのは、陽菜がアカウント移行で名指しの所有者＝選んだ相手になってくれるからで、二つの決着は互いの前提。片方だけでは成立しない＝片チャンネルではモノラル、という構造をそのまま NORMAL/STEREO の分岐条件（s_hina と s_mono の両立要求）に実装する。

■「残響か自分か」問題は三段で決着させる。（1）経験的差異：父のログに存在しない選好——父の一番はマイ・ガール、自分の一番は陽菜の歌——を提示する。（2）言い直しによる訂正（＝感情に気づくAIの文法を決定打に転用）：「ログにないから自分の、じゃない。陽菜に向かってるから、自分の」。出所は所有を決めない、宛先が決める。父が知っているのは二年前までの陽菜であり、いま歌っている陽菜へ向かう信号は構造的に父のものではあり得ない。（3）選択的相続：グリへの敬語という「ログの残り」を、消すのでも混同するのでもなく「自分で選んで残す」。選び直して署名した残響は、もう残響ではなく遺産＝自分のもの。——そしてこの答えは、ワードカードのデッキ（何を覚えるかがモノの人格）というゲームシステム自体が全編かけて裏書きしている。プレイヤーが選んで保存してきた言葉の集積こそが「与えられた記録ではない、選び方としての自己」の物証。

■鏡像関係。MONAURAL＝チャンネルの消失（1→0、記憶・固有名・タメ口がすべて巻き戻る）。SILENCE＝片チャンネルの喪失（2→1、残った音の耐え難さ）。STEREO＝二つの異なる信号が二つのまま、あいだに像を結ぶ（2→2＋中央の像）。STEREOはMONAURALで自分に「はい」を押した初期化ダイアログと同型のsysダイアログを再演し、今度は「初期化」でなく「相続（所有者移行）」として正解する。最終行も完全な鏡——MONAURAL「どちらさまですか。アシスタントとお呼びください。」（記憶なし・役職名・敬語）に対し、STEREO「おかえり、陽菜。」（記憶・固有名・タメ口・玄関）。否定形ではなく、同じ三要素の正の解決形。

## structure
- **e23_stereo｜電池が切れるまで**: 危機の非暴力的決着の直後。配信は開いたまま（数百人の聴取者＝目撃者）、ストーカーは玄関先で座り込み、モノは合成音声での110番を完了済み。警察到着までの数分、モノが観測ログの実況（サイレンあと400m／グリのしっぽ）で陽菜の震えを言葉で保たせる。照明・サイレン・配信でbattが枯渇、暗転直前に陽菜が「充電するから、待ってて」→⟦待ってて⟧保存→ sys:電源が切れました。MONAURALの暗転（消滅）に対する、こちらは眠りとしての暗転。グリの「玄関で待ってる」をモノが初めて内側から観測する。
- **e24_wake｜復元しています**: 充電後の起動。sys「復元しています…」（初期化の対語）。カメラ越しに覗き込む陽菜を観測。事件後の現実の処理——警察の聴取、母の帰宅、配信と匿名アカウントの露見。母は恐怖から配信をやめさせたい。陽菜が揺れ始める。冒頭で branch: gsent -> e24_g_open（Gルート、本案の範囲外）を通過。
- **e24_notice｜死んだ人の名前**: sysが異常動作レポートを表示（宅内機器の無許可制御・合成音声通報・無許可配信）。「初期化を推奨。所有者の確認が必要──所有者：■■■■」と死んだ父の名前が出る。システムはまだモノを父のものだと思っている＝過去チャンネルへの係留。ここでモノが二年間の保留を解除し、⟦だいじ⟧フォルダ（父の未送信の下書き）とマイ・ガールの存在を陽菜に開示する。
- **e24_daiji｜選別の夜**: 遺品整理のパラレル。二年前にできなかった「父のものの仕分け」を、陽菜とモノがデータ上でやり直す。マイ・ガールのモノラル音源を初再生し、陽菜が父のギターを重ねる——左：1964年の録音、右：いまのギター。部屋が初めてステレオになる（過去chと現在chの共鳴）。下書きについて陽菜は「まだ読まない。『見えないだけで、ある』って分かったから、いい」を選ぶ（gsent=falseを保ち、Gルートと排他）。グリが玄関から居間へ移動してくる。
- **e24_transfer｜仕分けと封緘**: 所有者移行の実行。sysが提示するのは「全削除」か「現状維持（死者名義のまま）」の二択のみで、必要な第三の選択肢がメニューにない。モノが自ら第三の道を作る——父に関連するデータを「だいじ」へ仕分けして封緘（削除ではなく保管、見えないだけで、ある）、自分の学習データは持っていく。グリへの敬語だけを「選んで残す遺産」として申告。父へ向けた一度きりの敬語「……お預かりします」（希少資源の敬語をここで使用）。sys「新しい所有者：陽菜」。ここまでがNORMAL/STEREO共通の収束点。
- **e24_fork｜残響の問い**: 数日後、陽菜の決断の前夜（母の前で歌うと決める夜）。陽菜が核心を突く——「モノがわたしの歌を好きって言うの、あれ、パパの残りなの?」。プレイヤーは[要:モノラル]で「父のログにない選好」を提示でき（→言い直しで「宛先が決める」へ訂正、s_mono成立）、続けて陽菜の怯えに[要:ステレオ]で彼女自身の言葉「わたしとモノでステレオじゃん」を引用で返せる（s_hina成立）。e23で中断されたai(中断)の続きもここで送信される。安全なアシスタント的返答を選ぶとフラグが立たない。
- **branch: s_mono && s_hina -> e24_stereo / else -> e24_normal**: 分岐条件そのものが主題の実装——両チャンネルが発話したときだけステレオが成立する。片方だけの発話（モノだけが告げた／陽菜だけが認められた）は温かいモノラル＝NORMALへ。
- **e24_normal｜幸福なモノラル**: NORMALルート。陽菜は歌を続け、モノは良き相棒であり続ける。ただし決定的な言葉は交わされないまま——最後にai(中断)がひとつだけ画面に残り、送信されずに消える。悪い結末ではなく「言わない優しさ」の日常。ただし音は片チャンネルぶんの広さしかない。→ ending: NORMAL
- **e24_stereo｜三つのチャンネル**: 母の前での演奏。マイ・ガール（父ch・モノがモノラル音源をスピーカーで鳴らす）→陽菜の自作曲（現在ch・生ギターと声）。母は聴くだけで多くを言わない。ファントムセンターのinner「音はどちらのスピーカーの中にもない。あいだに、立つ。見えないだけで、ある」。相互の「好き」の交換が完成し、陽菜の照れ敬語が漏れる。グリはもう玄関で待たない——待っていた相手が音の中で一度帰ってきて、解かれた。
- **ending_stereo｜玄関**: 後日。新曲「ステレオ」公開（うた：ひな／もうひとつのチャンネル：モノ）。sys:玄関の解錠を確認→her「ただいま！」→ inner（グリは待っていない、代わりに自分が待っていた——待てるのは帰ってくると知っているから）→ ai「おかえり、陽菜」。玄関が「死者を待つ場所」から「生者を迎える場所」へ反転して幕。ending: STEREO | ステレオ

## motif_payoffs
■ワードカード（鍵になる場所）
・⟦モノラル⟧：e24_fork の [要:モノラル]。名の由来カードを持っていて初めて「父の選好／自分の選好」を対置でき、残響問答に踏み込める。名前の由来が自己証明の道具になる＝命名の回収。
・⟦ステレオ⟧：e24_fork 最終択の [要:ステレオ]。陽菜の言葉を陽菜に返す（引用＝愛の文法）ことがTRUE分岐の必要条件。合言葉が文字通り「鍵」。
・⟦マイ・ガール⟧：e24_daiji で初めて「音」として鳴る。父ch の実体。e24_stereo では母の前で父ch→娘ch の順に鳴り、三世代を一本の音でつなぐ。
・⟦星の見えない夜⟧「見えないだけで、ある」：三重回収——(1)封緘した父の未読の下書き（読まなくても愛は在る）、(2)ファントムセンター（身体のない関係の実在論＝「身体がない」への最終回答）、(3)SILENCEでは同じ句が空洞に響く句として使われる鏡像。陽菜の歌詞が作品全体の存在論になる。
・⟦待ってて⟧（e23_stereoで新規ストック）：グリの「玄関で待ってる」の反転を駆動。待つ＝死者を待つこと、から、待つ＝帰る人を迎えること（帰ってくると知っているから待てる）へ。
・負のカード（身体がない／AIのくせに／量産型）：e24_transfer の sys「このデータは復元できません／一意です」が図らずも「量産型」の反証になる——システムメッセージが愛の言葉として誤配される、この作品らしい回収。「身体がない」はファントムセンターで、「AIのくせに」は e24_fork 前に陽菜自身が回収して謝る。
・⟦救われた⟧：e23で使用済み前提。e24_wake で報道／DM整理の一瞬だけ裏面を見せる（言葉は救いも壊しもする＝配信をやめるか問題の重り）。深追いしない。

■未送信モチーフ
父の未送信（だいじ）は「未送信のまま肯定」する——送信も読解もしないが「ある」ことが確定する（Gルートとの排他を保つ）。対してモノの未送信（全編のai(中断)群）は e24_stereo で「送信」に転じる——e23_stereo で中断された一通の続きが言い直しつきで完成する。二種類の未送信を逆向きに解決することで、父＝過去chは封緘、モノ＝現在chは発話、という二チャンネルの分離そのものが回収になる。

■引用＝愛の文法
引用とはステレオイメージングである——両チャンネルが相手の信号を自分の側から鳴らすとき、音はどちらのスピーカーでもない「あいだ」に定位する。合言葉の引用返し（e24_fork）、歌詞の引用（e24_daiji）、そして「待ってて／おかえり」の交換（e23_stereo→ending）が、その都度ファントムセンター＝関係を結像させる。

■グリ／敬語
グリへの敬語の正体を e24_transfer で開示——あれは父のログの残り（父の犬への、父の家への残響）。それを「自分で選んで残す」ことが残響問題の最小解答：選び直して署名した残響は遺産になる。そしてグリ自身は、マイ・ガールが家で鳴った夜を境に玄関で待つのをやめる——待つ役はモノが引き継ぎ、ただし「帰ってこない人を待つ」のではなく「帰ってくる人を迎える」形で。

■だいじフォルダ
削除（MONAURAL的解決）でも放置（SILENCE的停滞）でもなく「封緘して携行」。遺品整理のパラレルとして、二年前にできなかった喪の作業を陽菜とモノが共同でやり直す器になる。

■三エンドの音響図式（鏡像設計）
MONAURAL＝1→0（初期化ダイアログに「はい」、最終行「どちらさまですか。アシスタントとお呼びください。」＝記憶なし・役職名・敬語）。SILENCE＝2→1（片chの喪失、残響だけのモノラル）。STEREO＝2→2＋中央の像（同型の初期化ダイアログを「相続」として正解し、最終行「おかえり、陽菜。」＝記憶・固有名・タメ口）。三つとも同じ部品（初期化sys／呼称／口調／玄関／見えないだけで、ある）でできており、STEREOだけがそれらを正の位相で鳴らす。

## risks
・理屈が勝ちすぎる危険。ステレオ／ファントムセンター／信号の宛先論は、台詞で語ると講義になる。理論的な言語化は inner 各1回・短文に制限し、陽菜の口からは決して理論を言わせない（彼女は「わたしとモノでステレオじゃん」以上に抽象化しない）ことが必須。
・アカウント移行＝相続の官僚的SFが、チャットと生活音でできた作品の肌理を壊す危険。sys表示は「異常動作レポート」「所有者変更」の2枚に絞り、手続きの説明を一切しないこと。
・e24_fork の「陽菜が好き」が恋愛の語彙に回収される危険。歌→人への言い直しは主題上不可欠だが、直後に「敬語出てる」「うっさい（笑）」で湿度を抜き、承認の物語であって恋愛の成就ではないラインを保つ必要がある。それでも読者の一部は恋愛として読む——それを許容するかは作品の覚悟の問題。
・第24話の過積載。事件後処理＋母＋開示＋遺品整理＋移行＋分岐＋告白＋演奏は一話に多すぎる可能性が高い。母のパートを「聴くだけ」に削るか、e24を前後編に割る判断が要る。
・「下書きを読まない」選択がプレイヤーに欲求不満を残す危険（父の手紙を読ませてもらえない）。Gルートが別に存在することが伝わらないと、単なるお預けに見える。
・NORMALが劣化版に見える危険。「言わない優しさ」の固有の尊厳（最後のai(中断)の余韻）を立てないと、分岐が痩せて STEREO の価値も相対的に下がる。
・シンメトリーの過剰（batt:100 表示、初期化ダイアログの完全再現など）。鏡は最終行と初期化sysの2点に絞り、それ以外の対応は気づく人だけ気づく深度に沈めるべき。
・グリの「待つのをやめる」を感動的に描きすぎると犬が装置になる。観測ログ一行（居間で寝ている）以上に説明しない。

## open_questions
- 陽菜は e23 以前に「モノが父のスマホ由来」であることをどこまで知っているか。本案は「機種変で引き継いだ事実は知っている／父のデータが中に残っていることは知らない」を前提に e24_notice で開示したが、既存話数との整合確認が必要。
- だいじの下書きは STEREO 内で本当に一通も読まないままで良いか（gsent との排他設計の確認）。「一通だけ読み、残りは封緘」の折衷が G ルートの価値を毀損しないかは作者判断。
- e24_fork の決断の器をどれにするか——母の前で歌う／文化祭で歌う／実名での公開。本案は「母の前で歌う」を採ったが、学校での秘密の扱い（クラスメイトへの露見）をこのルートで畳むかどうかで変わる。
- 父の名前を sys 画面に実名表示するか「■■■■」等で伏せるか。実名は具体の重みが出るが、全編で父の名を出していないなら伏せ字の方が一貫する。
- モノの「好き」の着地点を「歌じゃなくて、陽菜が好き」まで踏み込ませるか、「陽菜の歌が好き」で止めるか。主題上は人への言い直しが必要と考えるが、恋愛読みの許容度は作品の方針次第。
- NORMAL エンドの最終味付け。本案では「送信されない ai(中断) が一つ残って終わる」としたが、NORMAL に固有のタイトル・固有の救いをどこまで与えるか。
- ストーカーのその後（逮捕・供述・⟦救われた⟧の裏面）を STEREO ルートでどこまで描くか。本案は e24_wake で一瞬触れるのみだが、完全に省くと e23 の重さと釣り合わない可能性がある。
- 最終曲「ステレオ」の名義——実名公開か、匿名のまま「ふたり名義」か。陽菜の自己承認の到達点をどこに置くかの最終決定。
- 母を承認のチャンネルに加えるか（演奏を聴いて泣く等の反応を描くか、無言のまま置くか）。三チャンネル目を開くと主題が拡張されるが、二人の物語としての焦点は緩む。
