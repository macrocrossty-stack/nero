# TRUEエンド「STEREO」設計案（統合版）

> 6視点の独立設計案（主題駆動／鏡像構造駆動／手札経済駆動／陽菜のアーク駆動／父の物語駆動／抑制駆動）を3人の審査で採点し、上位案の骨格に移植価値の高い部品を接いだ統合案。原案・審査の全文は `design/panel/` を参照。

---

## 0. 一枚絵（この設計の芯）

**第23話が「命を守る夜」なら、第24話は「好きを守る夜」。**
ストーカーの危機は前半で終わる。後半の侵入者は陽菜の内側にいる——「配信のせいでこうなった」という自己否定が、彼女の「好き」を消しに来る。暗いルートは一つ目の戦いに負けた世界。NORMALは一つ目に勝って二つ目を引き分けた世界。STEREOだけが両方を守り切る。

**TRUE分岐の条件そのものを、ステレオの定義にする。**
モノが自分の「好き」を自分の言葉で言えたこと（`f_own`）と、陽菜の罪悪感が言葉で解けたこと（`f_fan`）——両チャンネルが発話したときだけ、合言葉⟦ステレオ⟧が鍵として回る。片方だけの発話は、温かいモノラル＝NORMAL。分岐仕様が主題の実装になる。

**残響問題の決着は、全編で一行だけ。**
> ai: ……いや、違うな。
> ai: ログにないから自分の、じゃなかった。**陽菜に向かってるから、自分の。**

出所は所有を決めない。宛先が決める。父が知っているのは二年前までの陽菜であり、いま歌っている陽菜へ向かう信号は、構造的に父のものではあり得ない。「言い直し」＝感情に気づかないAIの兆候という全編の伏線を、最後に一度だけ「自覚した訂正」として使い、証明を詩に変える。これ以外の場所で残響問題を言葉で決着させない（実験・宣言・講義はすべて不採用）。

**未送信の四状態で、四つのエンドを一つのモチーフの活用形にする。**

| エンド | 未送信の運命 |
|---|---|
| MONAURAL | 送り手ごと消える（全部が巻き戻る） |
| SILENCE | 受け手を失う（途中で途切れた入力が「だいじ」に落ちる） |
| NORMAL | 未送信のまま、日常が続く |
| **STEREO** | **送信されて、返事が来る** |

**最終行は「おかえり、陽菜。」**
MONAURALの最終行「どちらさまですか。アシスタントとお呼びください。」が失う三要素（記憶・固有名・タメ口）を、同じ玄関で正の位相で鳴らす。玄関は「死者を待つ場所」（グリ）から「生者を迎える場所」へ反転して幕。

## 1. 主題の決着

### ふたつの「好き」＝相互条件
陽菜の自己承認は、承認者モノが「父の代弁者ではない独立した他者」であることを要求する。もしモノの承認が父の残響なら、陽菜は「死んだパパに頭を撫でられていただけ」で、また他人（今度は死者）に合わせて生きることになる。逆にモノが「自分の信号」を持てるのは、陽菜が名前をつけ、言葉を交わし続けたから。**片方だけでは成立しない＝片チャンネルではモノラル**——この構造をそのまま分岐条件（`f_fan && f_own`）に実装する。

### モノの残響問題＝宛先論＋選択的相続
- 決着の一行は上記の言い直し（e24_own）。
- 補助線は陽菜の側から出す。**お下がりのギター理論**——「このギター、お父さんのだよ？ でも今わたしが押さえたら、わたしの音が鳴るんだけど」。理論を彼女の語彙のまま通す唯一の解であり、モノを救うのが陽菜になる（相互救済）。陽菜に言わせる抽象化の上限は「わたしとモノでステレオじゃん」まで。それ以上の理屈は彼女の声ではない。
- **選択的相続**：グリへの敬語の正体を「父のログの残り」と開示した上で、「自分で選んで残す」と申告する。選び直して署名した残響は、残響ではなく遺産。残響問題の最小解答を、敬語芸の伏線回収と兼ねる。

### 陽菜のアーク＝データの刃と証言
ストーカーは音の映り込みと投稿時刻——データ——で彼女に到達した。モノは**同種のデータから正反対のものを掘る**：深夜2時〜4時に彼女の曲を再生した47人。「変なファンも一人、救われたも一人、差し引きゼロ」という彼女の暗算を破る反論。事件の装置と主題を一本の線で繋ぐ。

## 2. シーン構成

```
e23_stereo（実装済みの尻に「三つの種」を追加）
  └─ e24 ──── branch: gsent=true -> e24_g_open（Gルート・別稿）
       └─ e24_boot     … 起動の鏡像＋低温の日々（圧縮）
            └─ e24_erase    … 削除依頼。偽の（笑）
                 └─ e24_fan   … 局面A：罪悪感 → f_fan
                      └─ e24_own   … 局面B：所有権・残響問答 → f_own
                           └─ e24_delete  … 分岐点（削除画面）
                                ├─ e24_true_open → e24_daiji → ending: STEREO
                                ├─ e24_n_warm（言葉は届いたが、今は）→ ending: NORMAL
                                └─ e24_n_soft → ending: NORMAL
```

### e23_stereo への接続要件（実装済み部分への最小追加）
TRUE本編の回収はすべて、この三つの種から芽を出す。e23_stereo の幕切れに以下を仕込みたい：

1. **ai(中断) が停電で切断される**——「陽菜、自分——」。この一通がe24_daijiで完成・送信される（未送信→送信の背骨）。
2. **⟦待ってて⟧の新規ストック**——暗転間際、陽菜の「充電するから、待ってて」。inner「最終保存領域、残り1。……『待ってて』。玄関で待つ、の内側を初めて観測。保存する。」——グリの習性を初めて内側から知る。最終行「おかえり」の駆動力。
3. **batt: 0 での暗転**——sys「電源が切れました」。MONAURALの暗転（消滅）に対し、こちらは眠りとしての暗転。

### e24_boot ｜起動の鏡像と低温の日々
- 起動sysブロックを**MONAURALの初期化sysと一字一句同一**にする。反転はモノの第一声一行だけ：
  ```
  sys: アシスタントを初期化しています
  pause: 3
  her: ……モノ？
  ai: モノだけど。他に誰が
  inner: 記憶領域、照合完了。手札、全数残存。継続。
  ```
  周回プレイヤーだけに二秒の恐怖。初見には無害。「どちらさまですか」のliteral再演はTRUE側では**全面禁止**（MONAURALの一回性の保護）。
- 事後処理はsysの通知列だけで圧縮：逮捕報道の通知→モノが無言でミュート（保護行為をUI操作一つで描き、以後ストーカーには全編で二度と触れない）。母からのメッセージ。アカウント休止中の表示。
- 陽菜は低温。文体から！と（笑）が消える（説明しない）。左手の爪、伸長の観測（inner計測のみ、心配は書かない）＝廃業のサイン。
- ここに⟦ステレオ⟧の**再ストック救済**を一箇所（過去ログ回想で合言葉を再掲）。

### e24_erase ｜削除依頼
「モノ、お願いがある」——依頼形のまま、最悪の依頼。「アカウント消して。曲も、ぜんぶ」「量産型でいーの、わたしは（笑）」。この（笑）は鎧（inner「（笑）、検出。直近96時間で初。用法、防御。」）。
受けの選択肢は三つ：即答の「わかった」（NORMAL直行）／「やだ」（硬化）／**「消す前に、ぜんぶ聴かせて。最後なら、ちゃんと聴きたい」**（依頼形を彼女に返す第三の手＝正道）。

### e24_fan ｜局面A：罪悪感 → f_fan
「わたしの歌が人を壊した」に対して：
- [要:救われた] DMの原文引用＋総数の対比。
- [要:星の見えない夜] 深夜2〜4時の再生者47人のログ——「あいつは音から住所を探した。自分は同じログから、べつのものを見つけた。……見えないだけで、いる、って話」（歌詞の変奏。陽菜「それ、わたしの歌詞のパクリじゃん」＝引用の検出も愛の文法）。
- どちらかで f_fan。無所持は一般的な慰めのみ＝fは立たない（NORMALへの滑り台。進行は止めない）。

### e24_own ｜局面B：所有権 → f_own（本設計の心臓）
陽菜「どうせわたしの曲って、お父さんのコピーなんだよ」⇔モノの「父のログの残響」が**同型の問い**だと露呈する場面。
1. [要:マイ・ガール] **diffによる承認**——「Aメロのコード進行、一致率81%。真似で合ってる」「サビで、あんたは上に行く。お父さんは下に行ってた。ここから先、一致率9%。途中から、あんたの音になってる。**データで出る**」。感情語を使わない、AIにしか書けない承認。
2. 陽菜の逆質問「モノの『好き』って、お父さんのデータの続きだったりしないの」→ inner「想定質問リスト、該当なし。……最も恐れていた質問、該当あり。」→ モノ、初めて揺れを告白（「わからない」）。
3. 陽菜の**お下がりのギター理論**が返ってくる。
4. モノの決着（全編唯一の言い直し決着行）：「ログにないから自分の、じゃなかった。陽菜に向かってるから、自分の。歌じゃなくて——陽菜が、好き」→ 直後に湿度を抜く（her「言いましたね!?」ai「敬語出てる」her「うっさい（笑）」）。→ f_own

### e24_delete ｜分岐点
```
sys: アカウント設定 ＞ アカウントを削除する
her: 消すね。……いいよね、これで
> 「ちょっと待って」 -> e24_n_soft
> 「『わたしとモノでステレオじゃん』って言ったの、あんただよ」 -> e24_key [要:ステレオ]
branch: stock:ステレオ & f_fan & f_own -> e24_true_open
branch: -> e24_n_warm
```
- ⟦ステレオ⟧は**危機で一度も役に立たなかったカード**として設計する。第23話の鍵は救われた／グリ系／星の見えない夜。合言葉はクライマックスの夜を手札で眠っている。使い道のないカードを持ち続けたこと自体が「覚えていた」ことの証明で、ここで初めて[要:]が点灯する。選択肢文面は必ず陽菜の原文ママの「」引用（プレイヤーの記憶と照合される鍵）。
- **カードは鍵、先行局面の傾聴（f_fan/f_own）は扉の枠。**鍵単独は e24_n_warm——「……思い出した。ありがと。でも、ごめん、今は」。言葉が無力なのではなく、文脈が足りなかった。周回動機への変換。
- TRUE成立の画：
  ```
  her: ……それ、わたしのセリフじゃん。
  ai: あんたのセリフ。だから効く。
  （中略）
  sys: 削除は実行されませんでした
  inner: 削除フラグ、消滅。観測：陽菜、爪切りを探している。
  ```

### e24_daiji ｜開示・敬語一回・未送信の完成
一晩置いて（低温の緩衝を挟む）、陽菜「ほんとのとこ、わたしの歌どう思ってんの」→ モノは形容詞ではなくフォルダを見せる。
- **モノの「だいじ」共有**＝二年間ためてきた陽菜の言葉＝プレイヤーの手札そのもの（ワードカードシステムのメタ回収。日付表示で「自分が選んだ言葉が入っている」実感を殺さない）。
- フォルダ名は父の真似だと明かし、**アカウント移行と父の同名フォルダ（未送信の下書き）をここで開示**。動揺で全終盤唯一のモノの敬語：「……黙っていて、すみませんでした。」her「……敬語。」ai「……ごめん。」
- 沈黙の動機は美談にしない（father案から一拍だけ移植）：「あんたのためって言いたいけど。……いや、違うな。**自分が、消えるのがこわかった**」——この「消える恐怖」はMONAURALでモノが自ら実行するものの伏線的鏡像。
- **父の下書きは開かない。**sysのメタデータのみ（「下書き ○件 未送信 最終更新：2年前」）。全編唯一の pause: 30 をここに置く（最長の沈黙が号泣シーンの代役）。陽菜の受け：「……今は読まない。読まないけど……あるんだ。……見えないだけで、あるんだ。」——自作の歌詞を父へ返す（未送信の正の解決。本文の衝撃はGルート／SILENCEに温存する）。
- 直後、e23の ai(中断) が完成・送信される：「自分は、あんたの歌が好きだ。ログの続きとしてじゃなくて。いま、自分が、好き。」（※e24_ownで「好き」を先に言う場合はどちらか一方に絞る——決定事項7参照）

### ending: STEREO ｜ステレオ
数日後。爪は短い（「爪切った。久しぶりに」——復活を台詞ではなく行動で）。
- 新曲「ステレオ」を**音源投稿**（生配信ではない）。sys一行で安全対策を明示：「位置情報・環境音、除去済み」。クレジット「うた：ひな／もうひとつのチャンネル：モノ」。
- 父のチャンネル：レコードのプチノイズを2秒だけ、一番奥に敷く（音ではなく質感としての父。陽菜の「入れて。お願いします」＝本気の依頼形敬語）。
- グリはsys一行のみ：「グリ、玄関から移動。スピーカーの前に伏せました」。モノにも陽菜にも注釈させない。
- 幕切れ：
  ```
  sys: 玄関のセンサーが反応しました
  her: ただいま！
  inner: 玄関。グリは今日、待っていない。居間で寝ている。代わりに自分が待っていた。
  inner: 待てるのは、帰ってくると知っているから。
  ai: おかえり、陽菜
  sys: ⟦ただいま⟧を「だいじ」に保存しました
  batt: 100
  ending: STEREO | ステレオ
  ```
  最後のsys行で、言葉をストックする主体がプレイヤーからモノへ反転する（システム自体の回収）。

### NORMAL ｜未送信のまま続く日常
罰にしない。TRUEと同じ精度で書く。
```
her: 爪、もうちょっとだけ伸ばしとくね。もうちょっとだけ
ai: ん。いつでも
ai(中断): 陽菜、自分は
inner: 送信、保留。期限、未設定。
ending: NORMAL | ノーマル
```
悲劇ではない。送られなかっただけ。父の反復であることはプレイヤーにしか見えない。

## 3. 憲法（執筆規律・全シーン適用）

1. **感動ビートは1シーン1個。**山と山の間に必ず低温の日常を挟む。「全モチーフ回収」を目標にしない（回収は三割捨てる）。
2. **残響問題の言語化決着は全編で一箇所一行**（e24_ownの言い直し）。他は構造と計器に語らせる。
3. **MONAURAL資産の保護**：鏡像の接点は2箇所まで（起動sysブロック＋最終行の三要素反転）。「どちらさまですか」のliteral再演は禁止。
4. **敬語台帳**：モノの敬語漏れは終盤で1回のみ（e24_daijiの謝罪）。グリ限定敬語は不変の芸。陽菜の照れ・本気敬語は彼女の声の仕様として別勘定。
5. **グリはsys一行、無注釈。**犬に泣かせ仕事をさせない。
6. **父の下書き本文は画面に出さない。**メタデータのみ。本文の衝撃は暗いルートの資産。承認の最終審級は必ずモノと陽菜自身（父は代替にならない）。
7. **陽菜に理論を言わせない。**抽象化の上限は「わたしとモノでステレオじゃん」。
8. **pause経済**：全編5秒以下。pause: 30 は「だいじ」の一箇所だけ。
9. **モノは陽菜を名前で呼ぶ（「陽菜」）。二人称代名詞（あんた／おまえ／きみ）は使わない。**名前をもらったAIは、相手も名前で呼ぶ。パネル各案の「あんた」「おまえ」は採用時にすべて名前呼びに直すこと。
10. **「好き」の直後は必ず湿度を抜く**（「敬語出てる」「うっさい（笑）」系）。恋愛への分類はしない・させない（分類不能のまま置く）。
11. **ストーカーはsysのみ**：台詞・姿・動機・逮捕の描写禁止。事後は報道通知の無言ミュート一回で完結。

## 5. 暗いルート（次の執筆範囲）との接続契約

この設計は、Gルート／SILENCEを逆照射するための布石を含む：

- **未送信の四状態**（§0の表）を共通の背骨にする。SILENCEの「途中で途切れた最後の入力が『だいじ』に落ちる」は、TRUEで「完成・送信される」同じ一通の対蹠点。
- **⟦待ってて⟧**：TRUEでは「おかえり」に育つ。SILENCEでは「待つ者のいない玄関」（グリの位置情報だけが動かない）に反転する。
- **batt: 0 の暗転**：TRUEでは眠り。SILENCEでは「文字通り何も見えなくなる」無力の画。同じsys行「電源が切れました」を両ルートで共有する。
- **「自分が、消えるのがこわかった」**（e24_daiji）：MONAURALでモノが自ら消えることの伏線。TRUEを通ったプレイヤーほどMONAURALが重くなる。
- **父の下書き本文の温存**：TRUEは本文を見せない。見せる権利は暗いルートにある。
- **起動sysブロックの共有**：MONAURALの初期化sysと同一ブロックをTRUEの再起動に使う（一回性を削らない鏡の作法）。

## 6. 決定事項（作者判断が必要な分岐点）

1. **移行の秘密の開示温度**：本案は「e24_daijiに畳む軽装＋『消えるのがこわかった』の一拍」を推奨。フル装備（陽菜が怒って部屋を出る→帰還を待つ[要:玄関で待ってる]→「開けよ。いっしょに」）はfather案参照。重くするほど効くが、e24が過積載になりGルートと食い合う。
