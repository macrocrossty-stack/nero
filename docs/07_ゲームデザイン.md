# ゲームデザイン — 「やりごたえ」の設計

「読むだけのノベルゲーム」から「触って進める探索ADV」へ寄せるための方針。
重要な前提: **選択肢は、第5章の「乗っ取り」演出のためにも必須**。
プレイヤーが「自分で選んできた」という手応えの蓄積がなければ、選択権を奪われる恐怖は成立しない。

## 1. 選択肢の3系統

| 系統 | 内容 | 例 | 物語上の意味 |
|---|---|---|---|
| 🌱 思い出選択 | 正解のない風味選択。選んだ内容を記録し、**終章で一つずつ燃やす** | 今朝焼くパン／読み聞かせる本／湖に描く絵 | プレイヤーの選択そのものが「装填」になる |
| 💬 会話選択 | ヨルへの質問・応答。分岐は小さく合流するが、返答と獲得情報が変わる | 夜のヨルに聞くこと | キャラを掘る。周回時に別の選択肢を読む動機 |
| 👁 恐怖選択 | 影との遭遇時「目を閉じる／見る」。見ると描写が濃くなり、影が育つ | 襲撃シーン | 世界法則（恐れるほど育つ）の体感。第5章でこの選択が**勝手に選ばれる** |

- エンディング分岐は**作らない**（ループが物語の背骨のため）。やりごたえは分岐ではなく「密度と収集」で出す。
- 選択肢の間隔は「10分に1回以上」を目安に配置する。

## 2. 探索パート

- **第3章・屋敷探索**: 行き先メニュー（使用人の部屋／肖像画の廊下／屋根裏…）から選んで調べる形式。全箇所を回ると次へ進む。訪問順で小さな台詞差分。
- **書庫の本棚**: 任意で本を読める（世界観テキストのアンロック＝ロア収集）。読んだ冊数は記録され、終章の「燃える記憶」が増える——**集めるほど、失うものが増える**設計。
- **第4章・村**: 話す相手と屋台を選ぶ形式。🌱買い物選択。
- 湖の星絵は将来的にミニインタラクション化を検討（星をクリックして繋ぐ）。当面は🌱選択で代替。

## 3. 収集要素（周回動機）

| 要素 | 内容 |
|---|---|
| 思い出のかけら | 🌱選択の記録。メニューから一覧できる（終章で焼失→2周目で「彼が憶えていてくれた」形で復元） |
| 書庫の本 | 読んだ本のロア図鑑 |
| 言い伝え | 村で聞いた話（2周目で真実の注釈がつく） |
| 回想 | スチル解放ギャラリー（将来実装） |

## 4. ボリューム見込み

| パート | 想定 |
|---|---|
| 1周目（プロローグ〜終章） | 3〜4時間（探索・選択肢込み） |
| 2周目（ヨル視点レイヤー） | +1.5〜2時間 |
| 3周目（8かいめのあさ） | +20〜30分 |
| 収集コンプ | +α |

同人ノベルADVとして十分な規模（参考: 高評価作の多くが4〜8時間帯）。
「ノベルゲームであること」は弱点ではなく、**最後にゲームの器ごと物語に加担させる**本作の仕掛けの前提条件。

## 5. スクリプトタグ仕様（エンジンv2）

```
[choice store=変数名 seed=1 opt1=まるパン opt2=編みパン opt3=木の実のパン]
  … 選択肢を表示。選んだ文字列を変数に保存。seed=1なら🌱記録にも追加。
  optNt=*ラベル を付けるとそのラベルへ分岐（省略時はその場で続行）。
  optNif=変数名 / optNhide=変数名 で表示条件（探索の「すべて回ったら次へ」等）。

[set var=変数名 val=値]        … 変数代入（フラグ）
[if var=変数名 eq=値] … [else] … [endif]   … 条件分岐（eq省略時は真偽判定）
```
