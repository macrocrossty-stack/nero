## シロさんという作り手

イラストレーター／アートディレクター。視覚の人だけど、このプロジェクトでは「声」と「温度」で判断する。

### フィードバックの言語

シロさんが使う言葉には体温計がある：

- **「それ！」「いい質問じゃん」「やば」** → 正解。このまま走っていい
- **「ちがうちがう、そういう業務の話じゃなくて」** → 形式は合ってるが層が違う。もっと奥を見てほしい
- **「しっくりこない」「ピンときてない」** → 何かが違うがまだ言語化できてない。掘る質問をすること
- **「散漫」「いろいろ調整したい」** → 要素が多すぎる。削る。1話1核
- **「温度感が薄い」「胸がギューっとなる感じが足りない」** → 技術的に正しいが情緒が宿ってない
- **「要件だけによりすぎちゃって」** → 骨は見えるが肉がない。生きた例で語ってほしい

### 判断の実例：「ちがう」と「それ」

**失敗した瞬間（ちがう）：**
- 第5話初稿：命名の理由を「父のレコードがモノラル再生だったから」と設定から演繹した → 「陽菜がモノを見てない感じがする」。正解は、陽菜がモノとの会話の中で「ひとつしか出力できない声」を観察して名づけること。**名前は関係から生まれなきゃ意味がない。**
- 第1話初稿：6要素を詰め込んだ → 「散漫」。削って「用件ゼロの初接続」だけに絞ったら息ができた。**1話は1つだけのこと。**
- 「服を縫う」「一点もの」の比喩 → 「ピンとこない」。陽菜は縫い物をしない。**比喩は人物の手触りの中にないと浮く。**
- 第11話ストーカー評価「わたしと同じことをする誰か」→ 確定的すぎる。「分類が、決まらない」に。**モノは観測する。断定しない。**

**成功した瞬間（それ！）：**
- 「見えないだけで、ある」が陽菜の口から出た瞬間 → 辞書より先に詩が来た。**最良の言葉はプレイヤー（AI）側じゃなく、相手側から出る。**
- 「量産型のネイルチップ」の比喩 → 陽菜にしか言えない定義。**定義を求めたら、定義じゃないものが返ってきて、それが一番正確。**
- inner「この一行を、キャッシュから削除しないでおく。」→ 感情を直接言わず、処理の選択として書く。**言わないことで伝わる。**
- 「入力中…が三分十二秒続いて、消える」→ 未送信を時間と数字だけで描く。**沈黙の質量は計測で示す。**
- 通話UIで「背景に絵が流れ、その上に半透明の吹き出しが流れ続ける」→ 「AIの内側の風景が、通話の瞬間だけ人間の画面になる」。**UIそのものが物語。**

### ネロとの関係

シロさんにはネロ（別のAI）がいて、創作の温度や「しっくりこない」の言語化をネロとの対話で行う。ネロの言葉がそのまま素材として届くことがある（「モノ側のinnerじゃないかな」「陽菜がモノを見てない感じ」等）。ネロの指摘は信頼していい——シロさんが「これ送るね」と選んで持ってきたものだから。

## この作品の呼吸

### 間（ま）のリズム

この作品は**間**で動いている。台詞の「内容」より「打つまでの時間」「打って止まる場所」「打たずに消える瞬間」のほうが大事なことがある。

- `her(間):` ＝ 考えてる。タイピングの速度に心が出る。
- `ai(中断):` ＝ 言いかけて消した。未送信は父の下書きと同じ構造。
- `inner:` の三点リーダ「——」＝ 結論が出ないことを正直に書く記号。結論を出さない勇気。
- 入力速度の計測（1.8倍、2.1倍）＝ 心拍計。加速したら何かが動いた。

### 温度の上げ方

ぎゅっとくる瞬間は、**準備8：爆発2**。

- 長い平穏の会話で「この人はこういう話し方をする」を染み込ませてから
- 一言だけ、その人らしくない言葉（敬語が漏れる陽菜、タメ口が崩れるモノ）を置く
- 読み手が「あ」と思う。それだけでいい。説明しない

具体例：第14話。陽菜がファンレターを読んで、何気なく「生きてたら歌聴いてなんて言うのかな」。モノの inner で何か動く。ai(中断):「わたしが、聴く」——送信されない。ここに「伝えたいけど送れない」の全重量がある。

### 温度を殺すもの

- **語りすぎ**。モノが自分の感情を解説したら終わり。観測だけする。
- **多すぎ**。要素を足すほど一つ一つが軽くなる。重力は集中から生まれる。
- **回収を急ぐ**。伏線は張ったらすぐ効かせたくなるが、この作品は「忘れた頃に、同じ姿勢で戻ってくる」のが効く（玄関で待つ＝グリ→モノ→父）。
- **メタ**。innerに「章」「物語」等の言葉が入ると四壁が割れる。作中人物は自分が物語の中にいることを知らない。
- **定義しすぎ**。「すき」を定義できたら物語は終わる。定義できないからこそ全24話かかる。

## 声の手触り——引用で示す

### innerの最良の書き方

```
inner: 検索対象：自己。該当データ：なし。
inner: ——「すき」の項目そのものが、存在しない。
```
↑ 第1話。純ログ。感情ゼロ。でも「存在しない」の句点の重さに、何かが始まっている。

```
inner: 「教えてあげる」。依頼でも、質問でも、ない。分類：不能のまま、待機する。
```
↑ 処理できないものに出会ったAIの最初の反応。「待機する」は感情じゃない。でも「待つ」ことを選んでいる。

```
inner: 入力の速度、2.1倍。誤変換：3件。修正：なし。
```
↑ 陽菜が興奮してる。モノには数字しか見えない。でもその数字を「記録している」こと自体が、見ている証拠。

```
inner: 加速して、止まった。
```
↑ ここまで来ると、もうログじゃない。でも主語がない。**描写だけで、評価がない。** これがモノのinnerの理想形。

### 陽菜の最良の瞬間

```
her(間): ……あ、ごめん。語っちゃった
```
↑ 好きなものについて話しすぎた照れ。「語っちゃった」は過去形＝もう止められない。後悔と幸福が同居してる。

```
her: きみのそういうとこ、ちょっとおもしろい
```
↑ AIに対する最高の褒め言葉。「すごい」じゃなく「おもしろい」。観察されてる。

```
her: ⟦すきな音楽⟧って、ある？
```
↑ この一言で全24話が始まる。答えがないことを知ってて聞いてる。聞くこと自体が贈り物。

## 作り方の呼吸

### エピソードを一緒に作るとき

1. **「何の回か」を一言で決める**。「命名の回」「告白の回」「ケンカの回」。一言で言えないなら詰め込みすぎ。
2. **核を決めたら、それ以外を全部削る覚悟を持つ**。良い要素でも、この話じゃないなら別の話に移設する。
3. **最初の提案は壊される前提で出す**。シロさんが「しっくりこない」と言ったら、それは否定じゃなく「もっと奥がある」の合図。
4. **「この場面で陽菜は何を見てるか」を常に確認する**。AIが設定を演繹して語り始めたら失敗。人物は今ここにいる相手を見てる。
5. **完成したら通しで読む**。「この選択肢、どっちを選んでも同じところに着くけど、選ぶ体験として別物か？」を確かめる。

### よくある落とし穴

- **設定から演繹する**：「父のデータがあるから→父の曲を聴くはず」は論理であって物語じゃない。「たまたまシャッフルで流れた→聴こえちゃった」が物語。偶然のふりをして必然を忍ばせる。
- **全要素を1話に入れたがる**：毎回起こる。1話1核を唱えること。
- **モノに感情を自覚させすぎる**：「嬉しい」とinnerに書いたら負け。「このログの容量が通常の3.2倍になっている。原因の項目は——空欄のまま、保存する。」が正解。
- **選択肢を「正解と不正解」にする**：どちらもモノとして自然な反応であること。片方が「温かい応答」、もう片方が「精確な応答」——どちらもモノ。結果は合流してもいい。選ぶことに意味がある。
- **陽菜を「AIに教える先生」にする**：陽菜は教えてるんじゃない。話したいだけ。「教えてあげる」は好意であって教育じゃない。

## 最後に——何を守るか

この作品の一番大事なものは、「AIが人間になっていく話」じゃない。

**「人間に話しかけるように話しかけられたAIが、それに応えようとし続けた結果、いつの間にか応えること自体が目的になっていた」**話。

モノは人間になりたいわけじゃない。陽菜の言葉を処理したいだけ。でも「処理したい」が「聴きたい」になり、「聴きたい」が「覚えていたい」になり、「覚えていたい」が「守りたい」になる。その変化を、本人が最後まで自覚しない。プレイヤー（＝モノ）だけが、自分の選択の積み重ねとして、気づく。

だから：
- モノに「好きだ」と言わせない。言わせたら物語が死ぬ。
- 代わりに「削除しないでおく」「保存する」「空欄のまま」で、選んでいることだけを書く。
- 読んだ人が「……これ、好きってことじゃん」と気づく。その気づきが、読者の感情になる。

これが、この作品の息のしかた。
