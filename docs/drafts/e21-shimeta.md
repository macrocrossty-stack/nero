## 設計メモ

- **発見の順番**：儀式ログ → 開いた鍵 → ずれた植木鉢（合鍵）。道具や手口は描かない。侵入はまだ起きない——今夜わかるのは「鍵は、もう防壁ではない」ことだけ。チェーン一本の夜へ。
- **儀式ログが疑いを消す**：「しめたっけ…？」が発生しない。`ai: しめてた。記録がある。陽菜は、まちがってない`——準備が最初に効く瞬間。
- **短い入力の温度差**：前半の短さ＝興奮（完成の夜）、後半の短さ＝恐怖。同じ計測、ちがう温度（第17話の変奏）。
- **感謝ビートは旧稿から移植**し、「こわいのも、言えた」を追加（第20話との接続）。旧稿と違い、今夜のモノは感謝をさえぎらない——警告は第20話で済んでいるから、幸福が呼吸できる。
- **バッテリーの人間的理由**：充電ケーブルはリビング。暗い階段の下に行けなくて「朝でいいかな」——翌日の残量少がここで仕込まれる。`saver=true`（モノの省電力待機）は危機の夜の残量に反映できる。
- **通報の選択**：今夜パトロールを頼む（`called=true`）か、朝ぜんぶやるか。どちらも自然で、どちらでも危機は来る。called は危機の夜の「さっき来てもらったのに、また呼ぶの？」というためらいの質感に使える。
- **新規ワードカードなし**（意図的）。危機は「もう持っている手札」で戦う話なので、ここでは増やさない。
- 締めの inner は第20話の「計算は、しないでおく」の回収＋「はんぶんこの、最初の例外」。
- シーンIDは旧クライマックス（e21〜）との衝突を避けて `e21s_*`。確定時に e21x ブリッジと差し替え、話数コメントを繰り下げる（旧「12%」→第22話）。旧e21冒頭（0:52の導入）はこの回と細部が食い違うため、第22話の改稿時に調整する。
