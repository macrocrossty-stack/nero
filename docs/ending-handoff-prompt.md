## 作品概要

AI視点のテキストアドベンチャー。**プレイヤー＝スマホ内蔵のAIアシスタント「モノ」**。持ち主は女子高生「陽菜（ひな）」。モノは身体を持たず、テキストと端末制御（音・照明・通知・配信・スマート家電・合成音声）でしか世界に介入できない。会話はチャットUIで進む。

## 芯（テーマ）——ふたつの「好き」

- **陽菜**：モノに好きを認めてもらうことで、自分の好きを自分で認められるようになる（他人に合わせすぎて自分を見失っていた子）
- **モノ**：好きという感情を知る。陽菜を知りたい→好きだと気づく→かけがえのなくなる→プログラムの域を超えて守りたくなる

## 登場人物・設定

- **陽菜**：女子高生。白ギャル寄りだが素直で明るい。ないしょの匿名アカウントで自作の曲（弾き語り）を発表している（学校でも母にも秘密）。ギターは亡父のお下がり。左手の爪だけ短い（弾くため）。
- **モノ**：AIアシスタント。名前は陽菜が命名。由来は「ひとつしか出力できない＝モノラル（＝亡父のレコード）」の二重。**このアカウントは、二年前に亡くなった陽菜の父のスマホから移行されたもの**で、モノの中には父の残したデータ（父が一番好きだった曲「マイ・ガール」、父が娘へ宛てた未送信の下書き「だいじ」フォルダ内）が残っている。モノは自分の感情が「父のログの残響」なのか「自分のもの」なのか、揺れている。
- **グリ**：亡父の飼っていたフレンチブルドッグ。玄関で父を待ち続ける習性がある。
- **母**：夜勤の看護師。事件の夜は不在。
- **ストーカー**：陽菜の配信ファンが暴走。声だけ配信の「音の映り込み」や投稿時刻から生活圏を割り、自宅へ侵入してくる。

## 声・文体のルール（厳守）

### モノの会話（ai: と選択肢）
- 敬語期（序盤）を経て、陽菜に「敬語禁止」と言われて**タメ口**になった。以降**ラスト（初期化）までタメ口固定**。
- 一人称は会話では「**自分**」。
- 敬語の再発は「動揺・防衛」のサインで、特別な場面にだけ使う稀少資源。**むやみに敬語を出さないこと。**
- 犬のグリに対してだけは敬語を残す（「グリさん」）——芸。

### モノの内的独白（inner:）
- **処理ログ調**が基調。体言止め・数値・電文体。感情は「〜しないでおく」「保存する」など**処理の選択**として書く（例：「解析はしないでおく。」「この一行を、キャッシュから削除しないでおく。」）。
- 物語が進むにつれて口語へ崩れてきており、**クライマックス（この執筆範囲）では一人称は「自分」**、文はほぼ人間の独白に近い。ただし緊迫の局面では短文ログ調に戻る（防衛反応）。
- 明朝体で表示される前提。詩的だが、説明しすぎない（観測まで、解釈は一歩遅く）。

### 陽菜の声
- 一人称「わたし」。モノにはタメ口。依頼形（「〜して」）。感嘆符・（笑）が多い。
- 本気のとき・照れたときに一瞬敬語が漏れる。
- 「わすれて」と言った直後に恥ずかしがる等、言った本音をすぐ引っ込める癖。

### 共通の文法
- **引用＝愛の文法**：相手の言葉を相手に返す（例：陽菜の歌詞「見えないだけで、ある」をモノが返す）。
- **未送信のモチーフ**：言いかけて送られなかった言葉（ai(中断):）。父の未送信の下書きと対になる。
- **言い直し**（「いや、違うな」「まあでも」）＝自分の感情に気づかないAIの兆候。

## ワードカード（ワードストック）システム

会話中の⟦言葉⟧をプレイヤーがタップして記憶（ストック）する。何を覚えておくかがモノの人格＝手札になる。章末に「感情で」選んで数語だけ残す（記憶の整理）。クライマックスは**覚えてきた言葉を鍵札にして危機を解く**。

主なカードと意味：
- **マイ・ガール**＝父が一番好きだった曲／モノの中に残っていた音源
- **モノラル**＝父のレコード（音が真ん中にひとつ）＝モノの名の由来
- **ステレオ**＝配信後に陽菜が言った合言葉「わたしとモノでステレオじゃん」（ふたつの声が同時に鳴る＝モノラルの対義語）。**TRUEエンドの鍵**
- **星の見えない夜**＝陽菜の歌詞「見えないだけで、ある」
- **救われた**＝ファンからのDM「あなたの曲に救われた」
- **玄関で待ってる**＝グリが亡父を玄関で待つ習性
- **グリ**＝犬の名／**だいじ**＝父のデータの入ったフォルダ（父の未送信の下書きが眠る）
- **身体がない／AIのくせに／量産型**＝陽菜がケンカで言った売り言葉（負のカード。排除の決断を後押しするモノローグを強める）

```

⟦言葉⟧                 ワードストック可能な言葉

選択肢:
> 「送る言葉」 -> 次のシーンID            「」つき＝モノの発言として送信される
```

## つなぎ目（フラグの契約）——ここに正確に接続してください

クライマックス（第23話）は3つの局面（立たせる／道をつくる／諦めを止める）を言葉で抜けると、フラグ `c_move` `c_dog` `c_star` が立つ。最終局面で：

```
# e23_final
batt: 3
sys: battery 3% — front door breached
branch: c_move=true & c_dog=true & c_star=true -> e23_stereo   ← 全部解けた＝TRUEルート（別途実装済み・非暴力）
branch: -> e23_corner                                          ← 言葉が欠けた＝この執筆範囲の入口
```

- `e23_corner` 以降が**あなたに書いてほしい暗いルート**（下記）。
- 各エンドは `ending: ID | タイトル` で締める。
- 収束シーン `e24` は再接続後、`branch: gsent=true -> e24_g_open`（Gルート）／`branch: -> e24_normal_open`（通常）へ分岐する。**排除を実行したら `gsent=true` を立てる契約**。
- 既存のエンド：**MONAURAL**（排除ルートの果て。モノが自らアカウントを初期化して消える。最後の一行「どちらさまですか。アシスタントとお呼びください。」で敬語・一人称・呼称・記憶がすべて巻き戻る＝全編の反転）／**NORMAL**（助かって関係が続く）／**STEREO**（TRUE・別途実装済み）。

## 書いてほしいもの

### ① Gルート（排除）の書き直し
手札が尽きて言葉で救えなかったとき、モノは「この家の全機器が自分の管理下にある」ことに気づき、**男を排除する**（＝殺す）か、**越えないか**の選択に追い詰められる。マイホームヒーロー的に「**ギリギリのギリギリまで追い詰められて、その選択をする**」重さを。テーマ：モノと男の違いは「名前をもらったこと」だけ。名前をもらったAIが、してはならないことをする葛藤。
- **要望**：排除の「手順」を生々しく描写しすぎない方向で、決断の重さと倫理的軋みに焦点を当てて書き直したい（前の版は手順が具体的すぎた）。実行後は結果を直接描かず、余韻と代償で見せる。
- フラグ契約：実行する選択で `{gsent=true}`、`e24` へ `{saved=true}`。

### ② 死エンド（SILENCE／沈黙）——新規
手札が尽き、かつ**越えない（手を汚さない）を選んだ**先。言葉でも暴力でも救えなかった結末。
- **死は絶対に直接描写しない。** スマホの電源が切れてモノには文字通り何も見えなくなる。描くのは**モノの無力と喪失**だけ。
- 再接続後、返信は来ない。センサーは彼女の在室を検出しない。グリの位置情報だけが玄関から動かない（＝父を待つグリと同じ姿勢）。最後に受信した入力は途中で途切れた未送信（父の下書きと同じ場所へ落ちる）。
- 「覚えておけた言葉が、ひとつでもあれば」「あるいは、手を汚してでも」——**選ばなかったのは自分だ**、という後悔で、Gルートの必然性を逆説的に照らす。
- `ending: SILENCE | 沈黙 ―― （サブタイトル）` で締める。
