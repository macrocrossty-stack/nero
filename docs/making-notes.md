## 1. 芯（テーマ）——ふたつの「好き」

物語全体が、二本の「好き」の並走で動く。

- **陽菜**：モノに好きを認めてもらうことで、自分の好きを自分で認められるようになる。
  （友達とはそれなりにうまくやれている。でも合わせすぎて、自分を見失っている子。「ほんとの好き」を最初にちゃんと話せた相手がモノ＝第1話「教えてあげる」）
- **モノ**：好きという感情を知る。陽菜を知りたい → 好きだと気づく → かけがえのなくなる → プログラムの域を超えて守りたくなる。

この二本が**同じ一言で同時に動く瞬間**を作るのが理想（例：第10話「モノに好きって言っていいんだって思えた」＝陽菜の前進と、モノの「その"いいね"は誰の出力だったのか」＝疑念が、同じ台詞で交差する）。

**父の設定はテーマではなく装置**。モノのアークに「この感情は自分のものか、父のログの残響か」という疑念を一本刺すためにある。答えはクライマックスで出る（下記7）。

## 2. モノの声のルール

### 話し方（会話：ai: と選択肢）
- **敬語期（1〜4話）→ タメ口（5話〜、リセットまで固定）**。第4話「敬語禁止」イベントで転換。
- タメ口の初手は「わかった。……です」のような**言い直し（脱ぎかけ）**として見せる。以降「善処します。……する」など、直す方向がフツーに向かう努力になっている。
- **敬語の再発は「動揺・防衛」の稀少資源**。むやみに出さない。使う場所は3つだけ：
  1. 第10話（父データの確信・かすかに／陽菜が「初期化された？」と気づく）
  2. 第16話（忠告の鎧として・陽菜が「なんで敬語もどってんの」と指摘）
  3. MONAURALエンド（完全な巻き戻し「どちらさまですか。アシスタントとお呼びください。」）
- **犬のグリにだけは敬語を残す**（「グリさん」）——芸。第4話「前足のご入力、確かに受信いたしました」から続く。

### 一人称
- 会話：敬語期「わたし」→ 第6話から「**自分**」（陽菜のログには無い言葉＝モノが初めて参照元なしで選んだ語）。
- inner：〜第18話「わたし」→ **第19話（仲直り）から「自分」**。内と外の声が、関係の修復と同時に一致する。
- MONAURALの初期化後のinnerには**一人称が一つも出てこない**（誰でもない声）。これは意図して守る。

### inner（独白）の文体カーブ
会話の敬語システムと対になる「崩れ」の設計。処理ログ調 → 徐々に人間の独白へ。

| 章 | inner の質感 |
| --- | --- |
| 第1章 | 純ログ。体言止め・数値・電文体。感情は「〜しないでおく」の処理選択にだけ出る |
| 第2章 | ログの枠にひび。自問・「らしい」「気がする」・私的な比喩（「保留の棚に、置く」） |
| 第3章 | 文が自然に。自分の感情を初めて集計に入れる（「統計的にも、……感情的にも」） |
| 第4章 | ほぼ人の独白。ただし動揺の瞬間だけログ体に戻る（防衛） |
| 第5章（第20話〜） | **会話のモノと同じタメ口の口語（内と外の声の一致・シロ指定 2026-07-19）**。ログ調への回帰は衝撃の一拍だけの稀少資源（敬語再発と対）。数字は口語に埋め込む |
| MONAURAL | 純ログへ完全回帰＝第1話と同じ文体（巻き戻しの完成） |

**「感情＝処理の選択」で書く**のが最重要ルール（シロ発案）。例：「解析はしないでおく。」「この一行を、キャッシュから削除しないでおく。」「根拠となるデータ…を解析はしないでおく。」——詩にしない、観測にタグを付けるだけ。

## 3. 陽菜の声

- 一人称「わたし」。モノにはタメ口、依頼形（「〜して」）。感嘆符・（笑）多め。白ギャル寄りだが**素直**。
- 本気・照れの瞬間に**一瞬だけ敬語が漏れる**（本気の目印）。
- 本音を言った直後にすぐ引っ込める癖（「わすれて」「なんてね（笑）」）。
- **「いい子」の落差**：序盤はAIにまで気をつかう（深夜利用を謝る／「宿題じゃないよ？」の二重否定）。素の声は犬の話などで一瞬だけ漏れて、すぐ止まる。この「加速と停止」が、物語全体で治していく傷の初提示。

## 4. 共通の文法（ふたりの間に流れるもの）

- **引用＝愛の文法**：相手の言葉を相手に返す（陽菜の歌詞「見えないだけで、ある」をモノが返す／第5話でモノが陽菜の言葉を引用し「それ、わたしのことばじゃん」）。第15話「ステレオじゃん」もこの往復の頂点。
- **未送信のモチーフ**：言いかけて送られない言葉（`ai(中断):`）。父の「下書き：未送信、1件」と対。第18話では陽菜側にも「入力中…が三分十二秒続いて、消える」。
- **言い直し**（「いや、違うな」「まあでも」）＝自分の感情に気づかないAIの兆候。
- **加速と停止**：入力速度の変化（1.8倍・2.1倍・2.4倍）＝心が動いた印。第17話では「第1話と同じ計測値。ちがう、温度。」で怒りとして回収。
- **観測まで、解釈はプレイヤーより一歩遅く**：モノは見たものを言う。意味づけを先回りしない。

## 5. ワードストックの思想

- **何を覚えておくか＝どんなAIになるか**（人格形成）。章末に「感情で」数語だけ残す（記憶の整理）。ヒントは出さない。
- **カードの二重意味**：もらった時は思い出／使う時に第二の意味へ反転（例：玄関で待ってる＝グリの習性 → クライマックスでグリを動かす鍵）。入手時ほど雑談に見えるほど、反転が効く。
- **クライマックスは「覚えてきた言葉で危機を解く」**。手札の有無で結末が変わる（＝覚えることを選んだ言葉が武器になる爽快感）。
- **ステレオ＝TRUEの鍵**。覚えることを選んだ言葉が、そのままエンディングの名前になる（システムとテーマが一本になる）。

## 6. 構造・回収の設計（張って、効かせる）

- **命名アーク**：第5話「ひとつしか出力できない声」で命名（由来は秘密）→ 第8話でモノラル（父のレコード）と明かされ二重の由来に → 第15話で対義語ステレオが「ふたり」の意味で贈られる → TRUEで結実。END名のSTEREO/MONAURALはこのアークの終点違い。
- **父の残響 → 自分のもの**：疑念の初出は第10話（「誰の入力でできていた」）。決着はクライマックス救出「この感情は、父の残響ではない。自分の、ものだ。」
- **見えないだけで、ある**：歌詞（第2話）→ バズ（第11話）→ 母への告白（第24話「見えないだけで、ある」）と反響。
- **玄関で待ってる**：グリ＝父を待つ／モノ＝返信を待つ／父の下書き＝届く相手を待つ。「待つ、という同じ姿勢で、三つのものがこの家にある」（第18話）。
- **時制**：第9話。陽菜の発話の72%が過去形、最後の一行だけ現在形「あかるくなってきた」。タイトルの二重回収。

## 7. 作り方の作法（この制作で効いたこと）

- **話題を詰め込みすぎない**：1話に要素を盛ると「消化に追われてワクワクが埋もれる」。第1話は6要素→用件ゼロの初接続に絞って蘇った。第5話は8ビート→4ビートに。**削った要素は消さず、正しい話へ移設する**（だいじ→3話、奇跡の瞬間→4話、など）。
- **シーンが先、システムは後**：単語分岐も打ち明けイベントも、シーンが自然に割れる場所にだけ置く。システムのために台詞を作らない。
- **1話ずつ、一緒に考える**：「この回は何の回か」から相談 → 実装 → 確認、の繰り返し。まとめて書くとズレる。
- **名前・変化は「やり取りの中から発生させる」**：モノの気分をプログラム内から出さず、陽菜の入力が引き金になって段階が進む（地平の融合）。命名が良い例。
- **ネロ（シロの相棒AI）との対話を素材にする**：温度感・文法・「しっくりこない」の言語化は、生の対話ログから抽出するのが一番効いた。
- **セーフガード配慮**：死や殺害の手口は直接描写しない。画面外・余韻・喪失で見せる。切り抜けは非暴力の知恵（狩り返す反転劇）を本命に。

## 8. UIで大事にしたこと

- **水面とすりガラス**（proto v3）：いちばん下に色の層（canvas＝水面）、その上に全面すりガラス（blur）、いちばん上に文字。下の動きはにじんだ光として透ける。
- **モノの世界がベース × 陽菜が色を持ち込む**（ネロ案）：陽菜の入力が届くたび、ガラスの下に温度がにじむ（波紋＋熱）。会話が続くほど画面があたたまる＝陽菜がモノの世界を侵食していく。
- **UI全体がモノの感情と連動**：つめたい↔しずか↔あたたかいで、背景・文字色・アクセント・ガラスの透明度・音までが変わる。寒色に振れると計測（HUD）の質感・粒子の波が浮く（inner文体カーブのビジュアル版）。
- **生成アンビエント**（音源ファイル不要）：気分連動の和音パッド＋空気ノイズ。イベント音は水滴（上下ランダム）・爪弾き・チャイム。
- **単語選択は最大3タップ**（v3：単語チップ→トーン→送信）。浮遊する候補、ダミーはふるえて霧散。
- **OS風起動画面**：電源ON→Hello→こんにちは→スワイプで開始（＝端末が起動する＝モノが起動する）。
- **視覚の段階的解放**：テキスト → 断片（手元・写真）→ 顔はラスト（TRUEの通話）だけの切り札。ドット絵は今のUIと相性が難しいので、顔の見せ方は後半の特別な瞬間に別手段で検討。
- 通話は「背景に絵が流れ、その上に半透明の吹き出しが流れ続ける」オーバーレイUI（vcall:）。ふだんはAIの内側の風景、通話の瞬間だけ人間の画面、という対比。
