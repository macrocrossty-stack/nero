## 3. DSL仕様（確定本文で使用中の全記法）

### 行タイプ

| 記法 | 意味 |
|---|---|
| `// …` | コメント（各話冒頭に表裏テーマ・芯メモ） |
| `# id` | シーン（ノード）定義。遷移のターゲット |
| `her: 文` | 陽菜の発言 |
| `her(間): 文` | 陽菜の発言・**間をとって表示**（タイピングの遅れ＝心情） |
| `ai: 文` | モノの送信済み発言 |
| `ai(中断): 文` | モノが**入力しかけて消した**発言（送信されない。演出必須） |
| `inner: 文` | モノの内心（プレイヤーにだけ見える） |
| `inner(if:条件): 文` | 条件成立時のみ表示されるinner。例 `inner(if:saver=true):` `inner(if:stock:身体がない):` |
| `sys: 文` | システム表示（時刻 `—— 1:22 ——`、効果音 `—— カチ、リ ——`、♪ 楽曲、🔊 スピーカー出力、接続状態 等） |
| `img: 説明` | 画像表示。**現状すべてダミー**（シロさんが後日差し替え） |
| `batt: N` | バッテリーHUDを絶対値Nに更新（**第22話冒頭からHUD ON**。分岐間desync回避のため絶対値指定） |
| `pause: N` | N秒の間 |
| `branch: -> id` | 無選択遷移 |
| `consolidate: N` | 記憶整理（ワードストック整理演出）。章の節目に配置 |

### 選択肢

```
> ラベル -> id                          基本形
> ラベル -> id {flag=true, warmth+=1}   フラグ変異つき
> ラベル | 実際に送信される本文 -> id     ラベルと送信文が別（例：第22話「そっと起こす」）
> ラベル [要:ワード] -> id              ワードストックにそのワードがある場合のみ選択可能（ゲート）
```

- `⟦ワード⟧` は本文中でワードがストックに入る（＝プレイヤーの手札になる）マーカー。
- ゲートの確定使用例はまだ旧稿のみ（`[要:マイ・ガール]` `[要:グリ]` `[要:星の見えない夜]` `[要:救われた]` `[要:ステレオ]`）。新23話でも同型を使う予定。

## 8. 実装上の注意（物語側からの制約）

- **チャットUIに「場所」を出さないこと。** 新23〜24話のトリック（陽菜の居場所の誤認）は「テキストはどの部屋からでも打てる」というUIの性質そのものが土台。位置表示・部屋名ラベル等を追加すると仕掛けが壊れる。
- `ai(中断):` は「入力中表示→消える」の演出が必須（未送信は本作の中核モチーフ）。
- `her(間):` は表示前の間・タイピング速度差で表現（速度＝心拍計）。
- 第18.5話の `sys: —— めも.txt が、更新されました ——` はリアルタイム更新の演出（通知風）。
- 暗い結末（MONAURAL等）は手段を描かず「選択→暗転→時間経過」で処理（シロ確定 2026-07-19）。演出実装もこの前提で。
